## Setting Up Webhooks

Webhooks let Zefir push events like new messages or user joins to your endpoint. Configure them in your Zefir dashboard at `https://dashboard.example.com/integrations/webhooks`.

<Steps>
  <Step title="Create Webhook" icon="plus">
    Navigate to the Integrations tab. Click "New Webhook" and enter your endpoint URL, such as `https://your-webhook-url.com/zefir-events`.
  </Step>
  <Step title="Select Events" icon="settings">
    Choose events like `message.received` or `user.joined`. Test the configuration with the "Send Test" button.
  </Step>
  <Step title="Verify Payload" icon="check-circle">
    Handle the incoming POST request on your server. Zefir sends JSON payloads with event details.
  </Step>
</Steps>

<CodeGroup tabs="Node.js,Python">
  ```javascript
  const express = require('express');
  const app = express();
  app.use(express.json());

  app.post('/zefir-events', (req, res) => {
    const event = req.body;
    console.log('Zefir event:', event.type, event.data);
    // Process notification
    res.status(200).json({ received: true });
  });

  app.listen(3000);
  ```
  ```python
  from flask import Flask, request, jsonify

  app = Flask(__name__)

  @app.route('/zefir-events', methods=['POST'])
  def zefir_webhook():
      event = request.json
      print(f'Zefir event: {event["type"]} {event["data"]}')
      # Process notification
      return jsonify({'received': True}), 200

  if __name__ == '__main__':
      app.run(port=3000)
  ```
</CodeGroup>

<Callout kind="tip">
  Always validate the `X-Zefir-Signature` header using your webhook secret to prevent unauthorized requests.
</Callout>

## Best Practices for Secure Connections

- Use HTTPS endpoints only for webhooks.
- Rotate secrets regularly in your Zefir dashboard.
- Rate-limit incoming requests to avoid abuse.
- Log all events but avoid storing sensitive user data.

<Callout kind="alert">
  Never expose your webhook secrets in client-side code or public repositories.
</Callout>

These integrations unlock Zefir's full potential. Start with webhooks for quick wins, then expand to social and productivity tools.
